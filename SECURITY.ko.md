## 취약점 제보 방법

의심되는 취약점은 공개 이슈로 등록하지 말아주세요.

아래 정보를 포함해 비공개로 제보해주세요.
- 영향받는 버전
- 재현 절차
- 요청/응답 PoC (민감정보 마스킹)
- 기대 동작 vs 실제 동작
- 영향도 평가

권장 채널:
- GitHub Security Advisories (비공개 취약점 리포트)
- 사용 불가 시, 비공개 maintainer 연락 요청 시 본 정책을 함께 언급

## 대응 목표

- 최초 확인 응답: 영업일 기준 3일 이내
- 분류/트리아지 결정: 영업일 기준 7일 이내
- 수정 일정: 심각도 및 악용 가능성 기반

## 공개 원칙

- Coordinated disclosure를 선호합니다.
- 공개 전 합리적인 수정 시간을 보장해주세요.
- 요청이 없으면 제보자 크레딧을 제공합니다.

## 범위 주의사항

- 명시적으로 허가된 시스템만 테스트하세요.
- 실제 고객 시크릿/토큰은 제보에 포함하지 마세요.
- 로그 공유 시 자격증명, 쿠키, API 키, JWT를 마스킹하세요.
