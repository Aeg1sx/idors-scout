# 기여 가이드

[English](CONTRIBUTING.md) | 한국어

IDOR Scout에 기여해주셔서 감사합니다.

## 개발 환경 준비

1. 의존성 설치:
```bash
npm ci
```

2. 빌드:
```bash
npm run build
```

3. 테스트 실행:
```bash
npm test
```

## 기여 워크플로

1. 기능 브랜치를 생성합니다.
2. 변경 범위는 작고 명확하게 유지합니다.
3. 동작 변경이 있으면 테스트를 추가/수정합니다.
4. PR 전에 `npm run build`, `npm test`를 실행합니다.
5. PR에 아래 내용을 포함합니다.
- 문제 정의
- 접근 방법 요약
- 리스크 노트
- 테스트 근거

## 코딩 가이드라인

- TypeScript strict mode 관례를 따릅니다.
- CLI/설정 호환성을 깨는 변경은 마이그레이션 안내를 포함합니다.
- 보안 민감 로직은 암묵적 동작보다 명시적 동작을 우선합니다.
- 예시/로그/리포트에서 시크릿/토큰은 반드시 마스킹합니다.

## Pull Request 체크리스트

- [ ] 코드가 빌드된다
- [ ] 로컬 테스트가 통과한다
- [ ] 문서/설정 예시가 필요 시 업데이트되었다
- [ ] 자격증명/토큰이 커밋되지 않았다
- [ ] 하위 호환성 영향이 설명되었다

## 보안 이슈 제보
