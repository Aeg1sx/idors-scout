# Security Policy

English | [한국어](SECURITY.ko.md)

## Supported Versions

Security fixes are applied on the latest released version.

| Version | Supported |
| --- | --- |
| latest | Yes |
| older than latest | No |

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Send a private report with:
- affected version
- reproduction steps
- proof-of-concept request/response (redacted)
- expected vs actual behavior
- impact assessment

Preferred contact:
- GitHub Security Advisories (private vulnerability report)
- If unavailable, open a private maintainer contact request and mention this policy

## Response Targets

- Initial acknowledgment: within 3 business days
- Triage decision: within 7 business days
- Fix timeline: based on severity and exploitability

## Disclosure

- We prefer coordinated disclosure.
- Please allow reasonable time for a fix before public disclosure.
- We will credit reporters unless you request anonymity.

## Scope Notes

- Only test systems you are explicitly authorized to assess.
- Do not include real customer secrets or tokens in reports.
- When sharing logs, mask credentials, cookies, API keys, and JWTs.
